## 完整逐字稿

講了這麼多工具和實踐，最後我想用一個觀念來為今天的分享做個總結：**Infra Core 思維**。

簡單來說，就是這句話：**地基決定建築的高度**。

---

### 冰山模型

我們可以用「冰山模型」來理解 Monorepo 的價值。

**水面上**是開發者日常接觸的東西：

- UI 組件開發
- Business Logic 實現
- 新功能、新頁面

這些是我們每天在做的事情，是用戶看得到的產出。

**水面下**是 Monorepo 自動化處理的東西：

- Build System（構建系統）
- CI/CD Pipeline（持續整合與部署）
- Dependency Graph（依賴圖譜）
- Cache Management（快取管理）
- Code Generation（程式碼生成）

這些是用戶看不到的，但決定了團隊能跑多快、能走多遠。

---

### 理想狀態

理想狀態是什麼？

**開發者只需要專注在水面上的事情**——UI、Business Logic、新功能。

水面下的基礎設施，由 Monorepo 工具鏈自動處理。你不需要每天擔心構建會不會壞、快取有沒有失效、依賴有沒有衝突。

這就是 Monorepo 的核心價值：**把基礎設施的複雜度封裝起來，讓開發者專注在產品**。

---

### 工具選擇的本質

今天我們談了很多工具：Nx、Turborepo、Bun、Vite+、Oxlint...

但不管你選擇哪個工具，核心思維都是一樣的：

1. **終結工具鏈碎片化**
2. **讓開發者專注在產品**
3. **把基礎設施的複雜度封裝起來**

所以我認為，**前端工程師也需要具備 Infra 思維**。

你寫的每一行配置、定義的每一個邊界，都是在為未來的數百位開發者、為團隊的開發速度和穩定性打下基礎。

**Monorepo 就是你的地基。地基穩了，上面才能蓋得高。**

### 整體節奏

- 這是整場演講的「昇華」環節，語氣要從「技術分享」轉為「理念傳達」
- 節奏可以稍微放慢，讓聽眾有時間消化
- 冰山模型是視覺化的概念，可以搭配投影片圖示

### 重點強調

- **「地基決定建築的高度」**：這是核心金句，要講得有力
- **冰山模型**：水面上 vs 水面下的對比，讓聽眾理解 Monorepo 的隱藏價值
- **「開發者只需要專注在產品」**：這是 Monorepo 的終極目標

### 避免的陷阱

- 不要讓聽眾覺得「不用某個工具就是錯的」
- 強調「核心思維都一樣」，工具只是實現方式
- 避免過於說教，保持分享的語氣
